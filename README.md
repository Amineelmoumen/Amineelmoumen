## 👋 Qui suis-je ?

Je m'appelle **Amine El Moumen**, étudiant ingénieur en génie logiciel. Je ne me contente pas de faire fonctionner du code — je cherche à comprendre *pourquoi* une architecture tient dans le temps, et *comment* un produit reste maintenable quand l'équipe change, que les délais se resserrent, et que les exigences évoluent.

Ce qui m'anime : construire des systèmes clairs plutôt que des systèmes impressionnants sur le papier. Un bon logiciel, pour moi, c'est un logiciel qu'un autre développeur peut reprendre six mois plus tard sans avoir besoin de m'appeler.

## 🎯 Ma vision de l'ingénierie

> Le code le plus impressionnant n'est pas le plus complexe — c'est celui que personne ne remarque parce qu'il fait simplement ce qu'il doit faire, sans surprise.

Trois principes guident la façon dont je travaille :

| Principe | Ce que ça signifie concrètement |
|---|---|
| 🧱 **Architecture avant syntaxe** | Je réfléchis à la structure d'un projet avant d'écrire la première ligne de logique métier |
| 📖 **Documenter comme je code** | Un projet sans README clair est un projet à moitié terminé |
| 🔁 **Itérer, mesurer, corriger** | Je préfère livrer une version simple qui fonctionne plutôt qu'une version parfaite qui n'existe pas encore |
